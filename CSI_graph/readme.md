# WiFi受信状態のゆらぎ測定

M5Stackを使ってWi-Fi CSIという情報を取得し、そのI/QデータをLCDにリアルタイム表示するプログラムです。

Wi-Fiは普段、スマホやPCをインターネットにつなぐために使っています。

でも、少し見方を変えると、Wi-Fiの電波は「空間の状態」を通って届いています。

・人が部屋に入る  
・人がWi-Fiルータと受信機の間を横切る  
・椅子に座る  
・手を動かす  
・その場で静止する  

こういった変化があると、Wi-Fiの電波は反射したり、遮られたり、回り込んだりします。

つまり、Wi-Fiの受信状態を細かく見ることができれば、
「人が動いたとき、電波は本当に変わるのか？」
を観察できるはずです。

詳細はNOTEに記載しています。  
https://note.com/yuzu_monaka_/n/ne140b89c1651

# 仕様

M5Stack Basicで、Wi-Fi CSI値とRSSI値をLCDにリアルタイムでグラフ表示するプログラムです。

ハードウェア：M5Stack Basic  
開発環境：PlatformIO  
フレームワーク：Arduino  

# 使い方

1. M5StackとWi-Fiルータの位置を固定する  
2. 自分以外人がいない状態にする  
3. ボタンBを押して、無人状態の基準CSI取得を開始する  
4. 基準取得中に自分の体が電波に影響しないように、すぐに別の部屋へ移動する  
5. 基準取得完了の「ピッ」という音が鳴るまで待つ  
6. WiFiルータとM5Stackの間を横切る  
7. LCDのグラフを見る

## WiFi 

### SSDIとPassword

以下を書き換えてください。  
static const char* WIFI_SSID = "YOUR_WIFI_SSID";  
static const char* WIFI_PASS = "YOUR_WIFI_PASSWORD";  

### 接続先

2.4GHz Wi-Fi に接続してください。  
5GHz専用SSIDには接続できません。

## CSIが更新されない場合

次を確認してください。
| 症状                     | 確認点                         |
| ---------------------- | --------------------------- |
| `Waiting CSI...` のまま   | Wi-Fi接続できているか               |
| CSI countが増えない         | pingが通っているか                 |
| `esp_wifi_set_csi` が失敗 | Arduino ESP32 core側でCSIが有効か |
| グラフがほぼ動かない             | M5Stackとルータの距離・向きを変える       |
| ノイズが大きすぎる              | ルータに近すぎる/遠すぎる可能性            |


[注]  
CSIは環境依存が強いので、最初はM5StackとWi-Fiルータの距離を1〜3m程度にして、人がその間を横切るようにすると変化が見えやすいです。
